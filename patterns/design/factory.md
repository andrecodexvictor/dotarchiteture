# Design Pattern: Factory

## Propósito
Fornece uma interface para criar objetos em uma superclasse, mas permite que as subclasses alterem o tipo de objetos que serão criados.

## Quando usar
* Quando o código não sabe de antemão os tipos exatos dos objetos que deve instanciar.
* Para isolar a lógica de criação de objetos da lógica de negócio principal.
